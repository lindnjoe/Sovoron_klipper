Remote Display Setup Guide

Serve your printer's physical display over HTTP with touch support. Appears as a camera in Mainsail/Fluidd.

Prerequisites

Pillow (required for image conversion):

pip install Pillow

libdrm (only needed for HelixScreen / DRM backend):

sudo apt install libdrm2

1 Add to Klipper Config

Add this to your printer.cfg (or an included cfg file):

[remote_display]
port: 8092
touch_device: /dev/input/event0
#display_backend: auto
#drm_device:
#framebuffer_device: /dev/fb0
#stream_fps: 5
#bind: 0.0.0.0

Commented-out lines show defaults. Only touch_device typically needs to be changed.

2 Find Your Touch Device

List all input devices:

cat /proc/bus/input/devices

Look for your touchscreen — it will have ABS in its handlers. Example:

I: Bus=0003 Vendor=222a Product=0001 Name="ILITEK ILITEK-TP"
H: Handlers=event3

The eventN number is your touch device. Set it in config:

touch_device: /dev/input/event3

Quick shortcut

Find touch devices automatically:

grep -B5 'ABS' /proc/bus/input/devices | grep -E 'Name|Handlers'

3 Find Your Display Device

Auto mode (the default) works for most setups:

  • If HelixScreen is running → uses DRM automatically
  • Otherwise → uses /dev/fb0 (fbdev)

If auto mode works for you, skip to Step 4.

Option A: Standard Framebuffer (fbdev)

For displays that use /dev/fb0 (most SPI/DSI displays without HelixScreen):

# List framebuffer devices
ls /dev/fb*

# Check resolution
cat /sys/class/graphics/fb0/virtual_size

Set in config:

display_backend: fbdev
framebuffer_device: /dev/fb0

Option B: HelixScreen / DRM Displays

For displays driven by HelixScreen or other DRM/KMS applications:

# List DRM cards
ls /dev/dri/

Find the active DRM card

Run this to test each card and find which one has an active display:

for card in /dev/dri/card*; do
  python3 -c "
import os, ctypes
drm = ctypes.CDLL('libdrm.so.2')
class Res(ctypes.Structure):
    _fields_ = [('count_fbs',ctypes.c_int),
      ('fbs',ctypes.POINTER(ctypes.c_uint32)),
      ('count_crtcs',ctypes.c_int),
      ('crtcs',ctypes.POINTER(ctypes.c_uint32)),
      ('count_connectors',ctypes.c_int),
      ('connectors',ctypes.POINTER(ctypes.c_uint32)),
      ('count_encoders',ctypes.c_int),
      ('encoders',ctypes.POINTER(ctypes.c_uint32)),
      ('min_w',ctypes.c_uint32),('max_w',ctypes.c_uint32),
      ('min_h',ctypes.c_uint32),('max_h',ctypes.c_uint32)]
class Crtc(ctypes.Structure):
    _fields_ = [('crtc_id',ctypes.c_uint32),
      ('buffer_id',ctypes.c_uint32),
      ('x',ctypes.c_uint32),('y',ctypes.c_uint32),
      ('width',ctypes.c_uint32),('height',ctypes.c_uint32),
      ('mode_valid',ctypes.c_int)]
drm.drmModeGetResources.restype = ctypes.POINTER(Res)
drm.drmModeGetCrtc.restype = ctypes.POINTER(Crtc)
drm.drmModeGetCrtc.argtypes = [ctypes.c_int, ctypes.c_uint32]
drm.drmModeFreeResources.argtypes = [ctypes.c_void_p]
drm.drmModeFreeCrtc.argtypes = [ctypes.c_void_p]
fd = os.open('$card', os.O_RDWR)
res = drm.drmModeGetResources(fd)
if res:
    for i in range(res.contents.count_crtcs):
        c = drm.drmModeGetCrtc(fd, res.contents.crtcs[i])
        if c and c.contents.buffer_id and c.contents.mode_valid:
            print(f'$card: ACTIVE {c.contents.width}x{c.contents.height}')
            drm.drmModeFreeCrtc(c)
            break
        if c: drm.drmModeFreeCrtc(c)
    drm.drmModeFreeResources(res)
os.close(fd)
" 2>/dev/null
done

Example output:

/dev/dri/card1: ACTIVE 800x480

Set in config:

display_backend: drm
drm_device: /dev/dri/card1

DRM Permission Requirements

Klipper needs CAP_SYS_ADMIN to read DRM buffers. Without this, you'll get a "buffer handle not accessible" error.

sudo systemctl edit klipper

Add these lines:

[Service]
AmbientCapabilities=CAP_SYS_ADMIN

Then restart:

sudo systemctl daemon-reload
sudo systemctl restart klipper

4 Configure Mainsail / Fluidd Camera

First, find your Pi's IP address:

hostname -I | awk '{print $1}'

Add to moonraker.conf:

[webcam display]
enabled: true
service: iframe
stream_url: http://<YOUR-PI-IP>:8092/?embed
snapshot_url: http://<YOUR-PI-IP>:8092/snapshot.jpg
aspect_ratio: 800:480

Replace <YOUR-PI-IP> with your actual IP address (e.g. 192.168.1.100). Do not use localhost — it resolves to the browser's machine, not the Pi.

Set aspect_ratio to match your display resolution (e.g. 480:320, 800:480, 1024:600).

The ?embed parameter hides the title bar so the display fills the camera panel.

5 Verify It Works

Restart Klipper, then open in a browser:

http://<YOUR-PI-IP>:8092/

You should see your display with touch support. Check the Klipper log if something goes wrong:

grep remote_display ~/printer_data/logs/klippy.log | tail -20

Troubleshooting

Display shows stale/frozen image

If you see the boot splash instead of the live display, HelixScreen is likely using DRM while remote_display is reading from fbdev. Set display_backend: drm and specify your drm_device.

"buffer handle not accessible" error

DRM mode requires CAP_SYS_ADMIN. See the DRM permission instructions in Step 3.

Mainsail shows broken image icon

  • Make sure you're using your Pi's actual IP in stream_url, not localhost
  • Verify the module is running: curl http://<ip>:8092/snapshot.jpg -o /dev/null -w '%{http_code}'

Title bar showing in Mainsail camera panel

Add ?embed to your stream_url: http://<ip>:8092/?embed

Touch not working

  • Verify the device exists: ls -la /dev/input/event*
  • Klipper needs write access to the touch device. Check permissions or add the klipper user to the input group: sudo usermod -aG input klipper

Port already in use

Another process is using port 8092. Find it with ss -tlnp | grep 8092 and either stop it or change the port in your config.
