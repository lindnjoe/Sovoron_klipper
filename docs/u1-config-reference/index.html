AFC U1 Extended ConfigurationNew

This page documents the configuration sections added by the u1_AFC_Extra branch for ACE PRO, OpenAMS, and FPS (Filament Position Sensor) buffer hardware. These units extend the AFC system beyond the standard BoxTurtle / NightOwl / QuattroBox / HTLF / ViViD units.

Note

ACE and OpenAMS units are stepperless — they use their own internal firmware for filament movement instead of Klipper-controlled stepper motors. Lane sections for these units omit stepper pin configuration (step_pin, dir_pin, enable_pin).

[AFC_ACE unit_name] Section¶

The following options are available in the [AFC_ACE unit_name] section. These control the configuration of the AFC system when interfacing with Anycubic ACE PRO hardware. ACE units communicate over a serial connection and support two operational modes: combined (multiple slots share one toolhead path via a combiner) and direct (each slot feeds its own extruder independently).

Warning

ACE units require a virtual hub sensor. You must configure an [AFC_hub] section with switch_pin: virtual for the ACE hub, since there is no physical hub sensor on ACE hardware.

[AFC_ACE Ace_1]
serial_port: /dev/ttyACM0
#    Serial port for ACE PRO hardware. This parameter must be provided.
#    Use the output of `ls /dev/serial/by-id/` to find the correct port.
hub: ace_hub1
#    Hub that this unit belongs to. Must reference an [AFC_hub] section
#    with switch_pin: virtual.
extruder: extruder
#    AFC_extruder name that this unit feeds into.
mode: combined
#    Default: combined
#    Operational mode. "combined" means multiple ACE slots share one
#    toolhead path (must retract current slot before feeding new one).
#    "direct" means each ACE slot feeds its own extruder independently.
baud_rate: 115200
#    Default: 115200
#    Serial baud rate for ACE communication.
feed_speed: 60
#    Default: 60
#    Feed speed in mm/s. ACE firmware expects mm/s natively.
retract_speed: 50
#    Default: 50
#    Retract speed in mm/s.
feed_length: 500
#    Default: 500
#    Distance in mm from ACE unit to toolhead. Used for feed moves.
#    Can be overridden per-lane in [AFC_lane] sections.
retract_length: 500
#    Default: 500
#    Distance in mm to retract filament back to ACE. Usually matches
#    feed_length. Can be overridden per-lane.
unload_preretract: 50
#    Default: 50
#    Distance in mm for ACE rewind before cut, to tighten the spool
#    and prevent loose loops.
dist_hub: 200
#    Default: 200
#    Slot-to-hub/combiner distance in mm. Used for two-phase loading
#    (prep feeds to hub, load feeds hub-to-toolhead).
#    Calibrate with ACE_CALIBRATE_HUB. Can be overridden per-lane.
load_to_hub:
#    Default: <none> (inherits from AFC global)
#    Unit-level override for load-to-hub behavior.
#    Can also be overridden per-lane.
use_feed_assist: True
#    Default: True
#    Enable/disable extruder motor feed assist for all slots in this
#    unit. When enabled, the extruder motor helps pull filament during
#    loading. Can be overridden per-lane.
extruder_assist_length: 50
#    Default: 50
#    Distance in mm to advance with extruder motor during feed assist.
extruder_assist_speed: 300
#    Default: 300
#    Extruder assist speed in mm/min.
sensor_approach_margin: 30
#    Default: 30
#    Distance in mm before the expected sensor position to switch from
#    full-speed feeding to incremental sensor-checking mode.
sensor_step: 40
#    Default: 40
#    Distance in mm per check during sensor approach.
calibration_step: 50
#    Default: 50
#    Distance in mm per check during calibration.
max_feed_overshoot: 100
#    Default: 100
#    Extra distance in mm to try past feed_length before giving up.
auto_spoolman_create: False
#    Default: False
#    When True, automatically create filaments and spools in Spoolman
#    from RFID data read by ACE. When False, RFID data still matches
#    to existing filaments/spools but won't create new ones.
#    Can be overridden per-lane.
poll_interval: 1.0
#    Default: 1.0
#    Sensor polling interval in seconds for status and runout monitoring.

FPS Integration Settings

When the extruder uses an AFC_FPS buffer as pin_tool_start, the FPS ADC value serves as the toolhead sensor. These thresholds control when ACE considers filament to have reached the extruder.

# FPS (Filament Pressure Sensor) integration thresholds
fps_threshold: 0.9
#    Default: 0.9
#    FPS pressure threshold for toolhead sensor (0.0-1.0).
#    Above this value, the sensor is considered triggered.
fps_load_threshold: 0.65
#    Default: 0.65
#    Lower threshold used during active operations (loading/calibration).
#    The latch mechanism prevents false clears, so we can trigger earlier
#    to catch brief pressure spikes during filament engagement.
fps_delta_threshold: 0.15
#    Default: 0.15
#    Minimum jump between consecutive FPS readings for filament
#    engagement detection during active operations.
fps_confirm_count: 3
#    Default: 3 (minval: 1)
#    Consecutive ADC readings above threshold required before the sensor
#    is considered triggered. ACE pulsed feeding can cause brief spikes;
#    requiring multiple consecutive readings filters these out.
#    At ~100ms per ADC callback, 3 readings adds ~200ms confirmation delay.

ACE Lane Configuration¶

ACE lanes are configured with standard [AFC_lane] sections but use a slot-based unit reference. Stepper pins are not needed. Several unit-level parameters can be overridden per-lane.

[AFC_lane lane1]
unit: Ace_1:1
#    Unit:Slot reference. Slot numbers are 1-based in config
#    (converted to 0-based internally). ACE PRO has 4 slots.
hub: ace_hub1
extruder: extruder
feed_length: 2800
#    Optional: per-lane override for feed_length (mm).
retract_length: 2500
#    Optional: per-lane override for retract_length (mm).
dist_hub: 200
#    Optional: per-lane override for dist_hub (mm).
use_feed_assist: True
#    Optional: per-lane override for feed assist enable/disable.
auto_spoolman_create: False
#    Optional: per-lane override for auto Spoolman spool creation.

Tip

Run ACE_CALIBRATE_HUB LANE=lane1 to automatically calibrate feed_length and dist_hub for a lane. The calibration uses incremental sensor-based feeding to measure the exact distance from the ACE slot to the toolhead sensor.

[AFC_OpenAMS unit_name] Section¶

The following options are available in the [AFC_OpenAMS unit_name] section. These control the configuration of the AFC system when interfacing with OpenAMS (Open Automatic Material System) hardware. OpenAMS units communicate with their own firmware and use an AFC_FPS buffer for filament pressure sensing.

OpenAMS inherits base unit configuration options (hub, extruder, buffer, LED colors, speeds) from the afcUnit base class, the same as BoxTurtle/NightOwl. Below are configuration values specific to OpenAMS.

Info

OpenAMS units do not support TurtleNeck-style (stepper-based) buffers. Only AFC_FPS buffers are compatible. If a non-FPS buffer is configured, it will be automatically disabled.

[AFC_OpenAMS Oams_1]
hub: oams_hub1
#    Hub that this unit belongs to.
extruder: extruder
#    AFC_extruder name that this unit feeds into.
buffer: FPS_1
#    AFC_FPS buffer name. Only FPS buffers are supported for OpenAMS.
oams: oams1
#    Default: oams1
#    OpenAMS hardware name/instance identifier. Used to distinguish
#    multiple OpenAMS units.
stuck_spool_auto_recovery: False
#    Default: False
#    When True, a stuck spool detected during printing triggers an
#    automatic unload + reload + resume cycle instead of just pausing.
#    When False (default), the printer pauses for user intervention.
stuck_spool_load_grace: 8.0
#    Default: 8.0 (minval: 0.0)
#    Grace period in seconds after load before stuck spool detection
#    begins. Prevents false triggers during initial filament engagement.
stuck_spool_pressure_threshold: 0.08
#    Default: 0.08 (minval: 0.0)
#    FPS pressure threshold for stuck spool detection. When pressure
#    stays below this value during printing, the spool may be stuck.
engagement_pressure_threshold: 0.6
#    Default: 0.6 (minval: 0.0)
#    Maximum engagement pressure threshold. FPS readings above this
#    value during loading indicate successful filament engagement.
engagement_min_pressure: 0.25
#    Default: 0.25 (minval: 0.0)
#    Minimum engagement pressure threshold.
clog_sensitivity: medium
#    Default: medium
#    Clog detection sensitivity. Valid values: "low", "medium", "high".
#    Higher sensitivity triggers clog detection more aggressively.

OpenAMS Lane Configuration

OpenAMS lanes are configured like standard [AFC_lane] sections but, like ACE, do not require stepper pin configuration.

[AFC_lane oams_lane1]
unit: Oams_1:1
#    Unit:Slot reference for this lane.
hub: oams_hub1
extruder: extruder

[AFC_FPS buffer_name] Section¶

The Filament Position Sensor (FPS) is a software-based buffer driver that uses an ADC (analog-to-digital converter) pin to read filament pressure. Unlike TurtleNeck buffers that use physical advance/trailing switches, FPS uses continuous analog readings to determine filament buffer state and adjust extruder rotation distance multipliers for smooth printing.

FPS buffers are used by ACE and OpenAMS units, and can also be used with stepper-based units as an alternative to TurtleNeck buffers.

Note

FPS has PSF (Pressure Sensor Filament) compatibility aliases: neutral_point maps to set_point, max_tension maps to low_point, and max_compression maps to high_point. You can use either naming convention.

ADC Configuration

[AFC_FPS FPS_1]
adc_pin: PA5
#    ADC pin for the FPS sensor. This parameter must be provided.
#    Use your MCU's ADC-capable pin (e.g. PA5, PC0, etc).
sample_count: 5
#    Default: 5
#    Number of ADC samples per measurement for noise averaging.
sample_time: 0.005
#    Default: 0.005
#    Time in seconds between ADC samples.
report_time: 0.100
#    Default: 0.100
#    ADC callback interval in seconds. Controls how often the FPS
#    value is updated.
reversed: False
#    Default: False
#    When True, reverses the ADC reading (1.0 - value). Use if your
#    sensor reads high when buffer is empty and low when compressed.

Buffer Tuning Parameters

# Buffer state thresholds
set_point: 0.5
#    Default: 0.5 (range: 0.1-0.9)
#    Target FPS reading — the ideal operating point. Buffer corrections
#    push the reading toward this value.
#    PSF alias: neutral_point
low_point: 0.1
#    Default: 0.1 (range: 0.0-0.5)
#    Threshold for buffer stretched / not feeding. Below this value,
#    the buffer is considered to be in tension (filament being pulled
#    faster than it's being supplied).
#    PSF alias: max_tension
high_point: 0.9
#    Default: 0.9 (range: 0.5-1.0)
#    Threshold for buffer compressed / filament loaded. Above this
#    value, the buffer is considered fully compressed.
#    PSF alias: max_compression

# Multiplier settings
multiplier_high: 1.15
#    Default: 1.15 (minval: 1.0)
#    Rotation distance multiplier when buffer is above set_point
#    (compressed state). Increases effective extrusion to relieve
#    buffer pressure.
multiplier_low: 0.85
#    Default: 0.85 (range: 0.0-1.0)
#    Rotation distance multiplier when buffer is below set_point
#    (stretched state). Decreases effective extrusion to let buffer
#    recover.
trailing_min_multiplier: 1.05
#    Default: 1.05 (minval: 1.0)
#    Minimum multiplier during trailing (compressed) state.

# Deadband and smoothing
deadband: 0.30
#    Default: 0.30 (range: 0.0-0.6)
#    Neutral window width centered on set_point. Within this band,
#    no correction is applied (multiplier stays at 1.0).
smoothing: 0.3
#    Default: 0.3 (range: 0.0-0.95)
#    Exponential moving average factor. Higher values give smoother
#    but slower response to changes.

# Timing
update_interval: 0.25
#    Default: 0.25 (minval: 0.05)
#    Correction timer interval in seconds. How often the multiplier
#    is recalculated based on the current FPS reading.
flip_cooldown: 180.0
#    Default: 180.0 (minval: 0.0)
#    Cooldown in seconds to suppress rapid direction changes between
#    advancing and trailing states.

# Error detection
filament_error_sensitivity: 0
#    Default: 0 (range: 0-10)
#    Error detection sensitivity. 0 = disabled. Higher values make
#    the buffer more aggressive at detecting filament errors.

# Misc
debug: False
#    Default: False
#    Enable debug logging for buffer state changes.
enable_sensors_in_gui: True
#    Default: (inherits from AFC global setting)
#    Show FPS buffer sensor in Mainsail/Fluidd GUI.

LED Settings

FPS buffers support LED indicators to show buffer state, similar to TurtleNeck buffers. A neutral state LED is also available (not present on TurtleNeck).

# LED configuration
led_index:
#    Default: None
#    LED index for buffer state indication. Reference a defined LED
#    in the [AFC_led] section.
led_buffer_advancing: 0,0,1,0
#    Default: 0,0,1,0
#    LED color (R,G,B,W) when buffer is expanding (advancing state).
led_buffer_trailing: 0,1,0,0
#    Default: 0,1,0,0
#    LED color when buffer is compressed (trailing state).
led_buffer_neutral: 0,0.5,0.5,0
#    Default: 0,0.5,0.5,0
#    LED color when buffer is in neutral/deadband zone.
#    (FPS-specific — not available on TurtleNeck buffers.)
led_buffer_disable: 0,0,0,0.25
#    Default: 0,0,0,0.25
#    LED color when buffer is disabled.

Configuration Hierarchy¶

Configuration values cascade through multiple levels. More specific settings override broader ones:

# ACE Configuration Hierarchy (most specific wins)
#
#   1. Per-lane override    [AFC_lane lane1] feed_length: 2800
#   2. Per-unit setting     [AFC_ACE Ace_1]  feed_length: 500
#   3. Hard-coded default                    feed_length = 500.0
#
# OpenAMS Configuration Hierarchy
#
#   1. Per-unit setting     [AFC_OpenAMS Oams_1] stuck_spool_load_grace: 10
#   2. Hard-coded default                        stuck_spool_load_grace = 8.0
#
# FPS Buffer — PSF Alias Resolution
#
#   neutral_point  -->  set_point     (neutral_point is checked first as default)
#   max_tension    -->  low_point     (max_tension is checked first as default)
#   max_compression --> high_point    (max_compression is checked first as default)

Complete Example — ACE Minimal Setup¶

A minimal working ACE PRO setup. The virtual hub is required because ACE has no physical hub sensor. The FPS buffer is optional but recommended for pressure-based toolhead sensing.

# =============================================================================
# ACE PRO Minimal Configuration
# =============================================================================

# --- Virtual hub (ACE has no physical hub sensor) ---
[AFC_hub ace_hub1]
switch_pin: virtual

# --- ACE unit ---
[AFC_ACE Ace_1]
serial_port: /dev/ttyACM0
hub: ace_hub1
extruder: extruder
feed_length: 500
retract_length: 500

# --- FPS buffer (optional — enables pressure-based toolhead sensing) ---
[AFC_FPS FPS_1]
adc_pin: PA5
set_point: 0.5

# --- Lanes (one per ACE slot, 4 slots max) ---
[AFC_lane ace_lane1]
unit: Ace_1:1
hub: ace_hub1
extruder: extruder

[AFC_lane ace_lane2]
unit: Ace_1:2
hub: ace_hub1
extruder: extruder

[AFC_lane ace_lane3]
unit: Ace_1:3
hub: ace_hub1
extruder: extruder

[AFC_lane ace_lane4]
unit: Ace_1:4
hub: ace_hub1
extruder: extruder

Complete Example — OpenAMS Minimal Setup¶

A minimal working OpenAMS setup. OpenAMS requires an FPS buffer for filament pressure sensing. The hub uses a physical sensor (not virtual). OpenAMS communicates with its own firmware — no stepper pins are needed in lane sections.

# =============================================================================
# OpenAMS Minimal Configuration
# =============================================================================

# --- Hub (OpenAMS uses a physical hub sensor) ---
[AFC_hub oams_hub1]
switch_pin: ^!mcu:HUB_PIN

# --- FPS buffer (required for OpenAMS) ---
[AFC_FPS FPS_1]
adc_pin: mcu:PA5
set_point: 0.5
low_point: 0.1
high_point: 0.9

# --- OpenAMS unit ---
[AFC_OpenAMS Oams_1]

extruder: extruder
buffer: FPS_1
oams: oams1

# --- Lanes (one per OpenAMS bay, 4 bays max) ---
[AFC_lane oams_lane1]
unit: Oams_1:1
hub: oams_hub1
extruder: extruder

[AFC_lane oams_lane2]
unit: Oams_1:2
hub: oams_hub1
extruder: extruder

[AFC_lane oams_lane3]
unit: Oams_1:3
hub: oams_hub1
extruder: extruder

[AFC_lane oams_lane4]

hub: oams_hub1
extruder: extruder

Tip

For stuck spool auto-recovery during printing, add stuck_spool_auto_recovery: True to the [AFC_OpenAMS] section. This makes OpenAMS automatically unload, reload, and resume instead of pausing for user intervention. Adjust stuck_spool_load_grace and stuck_spool_pressure_threshold if you get false triggers.

Complete Example — FPS Buffer Setup¶

A fully-tuned FPS buffer configuration with LED indicators. FPS buffers use an ADC pin to continuously read filament pressure and adjust the extruder rotation distance multiplier in real time. This example shows all tunable parameters at their defaults with explanations of what to adjust for common scenarios.

# =============================================================================
# FPS Buffer — Full Configuration Example
# =============================================================================

[AFC_FPS FPS_1]
# --- ADC hardware ---
adc_pin: mcu:PA5
sample_count: 5
sample_time: 0.005
report_time: 0.100
reversed: False

# --- Buffer state thresholds ---
#   Filament pressure: 0.0 = no pressure, 1.0 = max compression
#
#   |  low_point   set_point   high_point  |
#   |     0.1    [---0.5---]     0.9       |
#   |  TENSION    NEUTRAL     COMPRESSION  |
#
set_point: 0.5
low_point: 0.1
high_point: 0.9

# --- Multiplier tuning ---
#   When above set_point: multiplier_high speeds up extrusion
#   When below set_point: multiplier_low slows down extrusion
#   Deadband: no correction applied within this window of set_point
multiplier_high: 1.15
multiplier_low: 0.85
trailing_min_multiplier: 1.05
deadband: 0.30

# --- Response tuning ---
#   smoothing: higher = slower response, less jitter (0.0-0.95)
#   update_interval: how often corrections are applied
#   flip_cooldown: prevents rapid advance/trail oscillation
smoothing: 0.3
update_interval: 0.25
flip_cooldown: 180.0

# --- Error detection ---
#   0 = disabled, 1-10 = sensitivity (higher = more aggressive)
filament_error_sensitivity: 0

# --- LED indicators (optional) ---
led_index: AFC_Indicator:1
led_buffer_advancing: 0,0,1,0
#    Blue: buffer expanding, filament being pulled
led_buffer_trailing: 0,1,0,0
#    Green: buffer compressed, filament backing up
led_buffer_neutral: 0,0.5,0.5,0
#    Cyan: buffer in deadband, no correction needed
led_buffer_disable: 0,0,0,0.25
#    Dim white: buffer disabled

enable_sensors_in_gui: True
debug: False

Tuning Tips

Noisy readings? Increase smoothing (try 0.5-0.7) and sample_count (try 8-10).
Oscillating between advance/trail? Widen deadband (try 0.4) or increase flip_cooldown.
Sluggish correction? Lower smoothing (try 0.1-0.2) and update_interval (try 0.10).
Sensor reads backwards? Set reversed: True.
Using PSF naming? Use neutral_point / max_tension / max_compression instead of set_point / low_point / high_point.
